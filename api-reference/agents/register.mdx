## Overview

Registers a new agent in the TalkOS system, enabling them to receive transferred calls from AI assistants. Agents can be configured with specific skills, availability settings, and capacity limits.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your tenant identifier
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Body Parameters

<ParamField body="agentId" type="string" required>
  Unique identifier for the agent. Must be alphanumeric with underscores, 3-50 characters.
</ParamField>

<ParamField body="name" type="string" required>
  Display name for the agent (2-100 characters)
</ParamField>

<ParamField body="email" type="string">
  Agent's email address for notifications
</ParamField>

<ParamField body="phoneNumber" type="string" required>
  Agent's phone number in E.164 format for receiving transferred calls
</ParamField>

<ParamField body="skills" type="array" required>
  List of skill tags the agent possesses for routing purposes
</ParamField>

<ParamField body="maxConcurrentCalls" type="integer" default="1">
  Maximum number of calls the agent can handle simultaneously (1-10)
</ParamField>

<ParamField body="priority" type="integer" default="5">
  Routing priority for this agent (1-10, higher = more priority)
</ParamField>

<ParamField body="languages" type="array">
  Languages the agent can support (ISO 639-1 codes)
</ParamField>

<ParamField body="timezone" type="string">
  Agent's timezone for scheduling (IANA timezone format)
</ParamField>

<ParamField body="workSchedule" type="object">
  Agent's working hours
  <Expandable title="properties">
    <ParamField body="monday" type="object">
      Monday schedule with `start` and `end` times (HH:MM format)
    </ParamField>
    <ParamField body="tuesday" type="object">
      Tuesday schedule
    </ParamField>
    <ParamField body="wednesday" type="object">
      Wednesday schedule
    </ParamField>
    <ParamField body="thursday" type="object">
      Thursday schedule
    </ParamField>
    <ParamField body="friday" type="object">
      Friday schedule
    </ParamField>
    <ParamField body="saturday" type="object">
      Saturday schedule
    </ParamField>
    <ParamField body="sunday" type="object">
      Sunday schedule
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="metadata" type="object">
  Custom key-value pairs for additional agent data
</ParamField>

<ParamField body="initialStatus" type="string" default="offline">
  Initial status: `available`, `offline`
</ParamField>

<ParamField body="notificationPreferences" type="object">
  How the agent receives transfer notifications
  <Expandable title="properties">
    <ParamField body="sms" type="boolean" default="true">
      Receive SMS notifications
    </ParamField>
    <ParamField body="email" type="boolean" default="false">
      Receive email notifications
    </ParamField>
    <ParamField body="webhook" type="string">
      Webhook URL for notifications
    </ParamField>
    <ParamField body="pushEnabled" type="boolean" default="false">
      Enable push notifications
    </ParamField>
  </Expandable>
</ParamField>

## Skills

Skills are used for intelligent call routing. Common skill categories include:

| Category | Examples |
|----------|----------|
| Department | `billing`, `sales`, `support`, `retention` |
| Technical | `technical_support`, `enterprise`, `developer` |
| Product | `product_a`, `product_b`, `enterprise_plan` |
| Language | Use the `languages` field instead |
| Specialty | `escalations`, `vip`, `complaints` |

## Best Practices

1. **Unique Agent IDs**: Use a consistent naming convention like `agent_firstname_number`
2. **Skill Tagging**: Be specific with skills for better routing accuracy
3. **Capacity Planning**: Set `maxConcurrentCalls` based on agent capability and call complexity
4. **Priority Levels**: Use higher priority (8-10) for senior agents or specialists
5. **Timezone Accuracy**: Set correct timezone for accurate schedule enforcement

## Error Codes

| Code | Description |
|------|-------------|
| `INVALID_AGENT_ID` | Agent ID format is invalid |
| `AGENT_ALREADY_EXISTS` | An agent with this ID exists |
| `INVALID_PHONE_NUMBER` | Phone number is not valid E.164 format |
| `PHONE_NUMBER_IN_USE` | Phone number registered to another agent |
| `INVALID_EMAIL` | Email format is invalid |
| `INVALID_SKILLS` | Skills must be a non-empty array of strings |
| `INVALID_CONCURRENT_CALLS` | Must be between 1 and 10 |
| `INVALID_PRIORITY` | Priority must be between 1 and 10 |
| `INVALID_TIMEZONE` | Timezone format not recognized |
| `INVALID_SCHEDULE` | Work schedule format is invalid |
