# Enroll Voice Print

Create a new voice biometrics profile to enable voice-based identity verification. This is the first step in setting up voice authentication for a caller.

## Endpoint

```
POST /api/voice-biometrics/enroll
```

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your unique tenant identifier
</ParamField>

## Request Body

<ParamField body="callerId" type="string" required>
  The caller's phone number in E.164 format (e.g., +14155551234)
</ParamField>

<ParamField body="callerName" type="string" required>
  The full name of the caller for identification purposes
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the enrollment was initiated successfully
</ResponseField>

<ResponseField name="voicePrintId" type="string">
  Unique identifier for the voice print profile
</ResponseField>

<ResponseField name="callerId" type="string">
  The enrolled caller's phone number
</ResponseField>

<ResponseField name="callerName" type="string">
  The enrolled caller's name
</ResponseField>

<ResponseField name="status" type="string">
  Current enrollment status: `pending`, `enrolled`, `active`
</ResponseField>

<ResponseField name="samplesRequired" type="number">
  Number of voice samples still needed for full enrollment
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp when the profile was created
</ResponseField>

## Notes

<Info>
After enrollment, you need to add at least 3 voice samples using the [Add Voice Sample](/api-reference/biometrics/add-sample) endpoint before the profile becomes active for verification.
</Info>

<Warning>
Ensure you have the caller's consent before enrolling them in voice biometrics. This may be required for regulatory compliance.
</Warning>
