# Verify Voice

Verify a caller's identity by comparing their voice from an active or completed call against their enrolled voice print profile. Returns a confidence score.

## Endpoint

```
POST /api/voice-biometrics/verify
```

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your unique tenant identifier
</ParamField>

## Request Body

<ParamField body="callerId" type="string" required>
  The caller's phone number in E.164 format (e.g., +14155551234)
</ParamField>

<ParamField body="callUuid" type="string" required>
  Unique identifier of the call to verify the voice from
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the verification was performed successfully
</ResponseField>

<ResponseField name="verified" type="boolean">
  Whether the voice matched the enrolled profile
</ResponseField>

<ResponseField name="confidence" type="number">
  Confidence score from 0.0 to 1.0 indicating match strength
</ResponseField>

<ResponseField name="callerId" type="string">
  The caller's phone number that was verified
</ResponseField>

<ResponseField name="callUuid" type="string">
  The call UUID used for verification
</ResponseField>

<ResponseField name="threshold" type="number">
  The minimum confidence threshold for a positive match
</ResponseField>

<ResponseField name="verificationId" type="string">
  Unique identifier for this verification attempt
</ResponseField>

<ResponseField name="verifiedAt" type="string">
  ISO 8601 timestamp when verification was performed
</ResponseField>

## Confidence Score Interpretation

| Score Range | Interpretation |
|-------------|----------------|
| 0.95 - 1.00 | Very high confidence - Strong match |
| 0.85 - 0.94 | High confidence - Likely match |
| 0.70 - 0.84 | Medium confidence - Possible match |
| 0.50 - 0.69 | Low confidence - Unlikely match |
| 0.00 - 0.49 | Very low confidence - No match |

## Notes

<Info>
The default verification threshold is 0.85. Scores above this threshold result in `verified: true`. You can adjust this threshold in your tenant configuration.
</Info>

<Warning>
Voice verification should be used as part of a multi-factor authentication strategy, not as the sole method of identity verification.
</Warning>

<Tip>
For higher security scenarios, consider requiring multiple successful verifications or combining with other authentication methods.
</Tip>
