# Batch Start Calls

Initiate multiple AI-powered outbound calls in a single request. Ideal for marketing campaigns, appointment reminders, surveys, and bulk notifications. Calls are queued and executed based on concurrency limits.

## Endpoint

```
POST /api/calls/batch/start
```

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your unique tenant identifier
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Request Body

### Required Fields

<ParamField body="calls" type="array" required>
  Array of call objects to initiate. Each call object supports all fields from the Start Call endpoint:

  <Expandable title="call object properties">
    <ParamField body="to" type="string" required>
      Phone number to call in E.164 format (e.g., `+14155551234`)
    </ParamField>

    <ParamField body="from" type="string" required>
      Your caller ID number in E.164 format
    </ParamField>

    <ParamField body="prompt" type="string" required>
      AI system prompt defining the agent's behavior
    </ParamField>

    <ParamField body="greeting" type="string">
      First message the AI speaks when call connects
    </ParamField>

    <ParamField body="webhookUrl" type="string">
      URL for individual call completion callback
    </ParamField>

    <ParamField body="maxDuration" type="number" default="600">
      Maximum call duration in seconds
    </ParamField>

    <ParamField body="language" type="string" default="en-US">
      Language code for the call
    </ParamField>

    <ParamField body="voiceId" type="string">
      ElevenLabs voice ID for text-to-speech
    </ParamField>

    <ParamField body="metadata" type="object">
      Custom metadata for the individual call
    </ParamField>

    <ParamField body="variables" type="object">
      Dynamic variables to inject into prompt (e.g., `{customerName}`, `{appointmentDate}`)
    </ParamField>
  </Expandable>

  Maximum 1000 calls per batch request.
</ParamField>

### Optional Fields

<ParamField body="campaignId" type="string">
  Unique campaign identifier for grouping and tracking batch calls. If not provided, a UUID will be automatically generated.
</ParamField>

<ParamField body="campaignName" type="string">
  Human-readable name for the campaign
</ParamField>

<ParamField body="priority" type="number" default="5">
  Queue priority for the batch (1-10). Higher priority batches are processed first. Default is 5.
</ParamField>

<ParamField body="maxConcurrent" type="number">
  Maximum number of concurrent calls from this batch. Must not exceed your plan's concurrency limit. If not specified, uses account default.
</ParamField>

<ParamField body="scheduleAt" type="string">
  ISO 8601 timestamp to schedule the batch for future execution. If not provided, batch starts immediately.
</ParamField>

<ParamField body="timezone" type="string" default="UTC">
  Timezone for scheduling (e.g., `America/New_York`, `Asia/Kolkata`). Used with `scheduleAt`.
</ParamField>

<ParamField body="callWindow" type="object">
  Time window restrictions for calling:
  - `startTime` (string): Earliest time to call in HH:MM format (e.g., `09:00`)
  - `endTime` (string): Latest time to call in HH:MM format (e.g., `18:00`)
  - `timezone` (string): Timezone for the window
  - `skipWeekends` (boolean): Skip Saturday and Sunday
</ParamField>

<ParamField body="retryPolicy" type="object">
  Automatic retry configuration for failed calls:
  - `enabled` (boolean): Enable automatic retries
  - `maxAttempts` (number): Maximum retry attempts (1-5)
  - `retryOn` (array): Conditions to retry on: `no-answer`, `busy`, `failed`
  - `delayMinutes` (number): Minutes to wait between retries
</ParamField>

<ParamField body="batchWebhookUrl" type="string">
  URL to receive batch completion callback when all calls finish
</ParamField>

<ParamField body="globalMetadata" type="object">
  Metadata applied to all calls in the batch
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the batch was successfully queued
</ResponseField>

<ResponseField name="batchId" type="string">
  Unique identifier for the batch. Use this to track batch status.
</ResponseField>

<ResponseField name="campaignId" type="string">
  Campaign identifier (provided or auto-generated)
</ResponseField>

<ResponseField name="totalCalls" type="number">
  Total number of calls in the batch
</ResponseField>

<ResponseField name="acceptedCalls" type="number">
  Number of calls accepted for processing
</ResponseField>

<ResponseField name="rejectedCalls" type="number">
  Number of calls rejected due to validation errors
</ResponseField>

<ResponseField name="rejections" type="array">
  Details of rejected calls:
  - `index` (number): Array index of rejected call
  - `to` (string): Phone number that was rejected
  - `reason` (string): Rejection reason
</ResponseField>

<ResponseField name="status" type="string">
  Batch status: `queued`, `scheduled`, `processing`, or `completed`
</ResponseField>

<ResponseField name="scheduledAt" type="string">
  ISO 8601 timestamp of scheduled execution (if scheduled)
</ResponseField>

<ResponseField name="estimatedCompletionTime" type="string">
  Estimated ISO 8601 timestamp when batch will complete
</ResponseField>

<ResponseField name="createdAt" type="string">
  ISO 8601 timestamp of batch creation
</ResponseField>

## Notes

- Maximum 1000 calls per batch request
- Calls are deduplicated by phone number within a batch
- DNC (Do Not Call) list is automatically checked before calling
- Variable substitution uses `{{variableName}}` syntax in prompts and greetings
- Batch execution respects rate limits and concurrency settings
- Use Get Batch Status endpoint to monitor progress
- Batch webhooks include aggregate statistics and individual call results
