# Get Queue Status

Retrieve real-time information about your call queue, including pending calls, active calls, and available capacity. Use this endpoint to monitor system load and optimize call scheduling.

## Endpoint

```
GET /api/calls/queue
```

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your unique tenant identifier
</ParamField>

## Query Parameters

<ParamField query="includeDetails" type="boolean" default="false">
  Include detailed information about queued and active calls
</ParamField>

<ParamField query="priority" type="number">
  Filter by specific priority level (1-10)
</ParamField>

<ParamField query="campaignId" type="string">
  Filter queue by specific campaign ID
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the request was successful
</ResponseField>

<ResponseField name="queue" type="object">
  Queue status object:

  <Expandable title="queue properties">
    <ResponseField name="pending" type="number">
      Number of calls waiting in the queue
    </ResponseField>

    <ResponseField name="active" type="number">
      Number of calls currently in progress
    </ResponseField>

    <ResponseField name="capacity" type="object">
      Capacity information:
      - `maxConcurrent` (number): Maximum concurrent calls allowed for your plan
      - `available` (number): Remaining concurrent call slots
      - `utilization` (number): Percentage of capacity in use
    </ResponseField>

    <ResponseField name="rateLimit" type="object">
      Rate limiting information:
      - `callsPerMinute` (number): Calls allowed per minute
      - `callsThisMinute` (number): Calls made in current minute
      - `remaining` (number): Calls remaining this minute
      - `resetsAt` (string): ISO 8601 timestamp when rate limit resets
    </ResponseField>

    <ResponseField name="estimatedWaitTime" type="number">
      Estimated seconds until next queued call starts
    </ResponseField>

    <ResponseField name="averageCallDuration" type="number">
      Average call duration in seconds (last 100 calls)
    </ResponseField>

    <ResponseField name="queuedByPriority" type="object">
      Breakdown of queued calls by priority level (1-10)
    </ResponseField>

    <ResponseField name="health" type="string">
      Queue health status: `healthy`, `degraded`, or `overloaded`
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pendingCalls" type="array">
  Details of pending calls (when `includeDetails=true`):

  <Expandable title="pendingCall properties">
    <ResponseField name="callUuid" type="string">
      Unique call identifier
    </ResponseField>

    <ResponseField name="to" type="string">
      Destination phone number
    </ResponseField>

    <ResponseField name="priority" type="number">
      Queue priority
    </ResponseField>

    <ResponseField name="campaignId" type="string">
      Campaign identifier (if part of batch)
    </ResponseField>

    <ResponseField name="queuedAt" type="string">
      ISO 8601 timestamp when call was queued
    </ResponseField>

    <ResponseField name="estimatedStartTime" type="string">
      Estimated ISO 8601 timestamp when call will start
    </ResponseField>

    <ResponseField name="position" type="number">
      Position in queue
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="activeCalls" type="array">
  Details of active calls (when `includeDetails=true`):

  <Expandable title="activeCall properties">
    <ResponseField name="callUuid" type="string">
      Unique call identifier
    </ResponseField>

    <ResponseField name="to" type="string">
      Destination phone number
    </ResponseField>

    <ResponseField name="status" type="string">
      Call status: `dialing`, `ringing`, `in-progress`
    </ResponseField>

    <ResponseField name="duration" type="number">
      Current duration in seconds
    </ResponseField>

    <ResponseField name="startedAt" type="string">
      ISO 8601 timestamp when call started
    </ResponseField>

    <ResponseField name="campaignId" type="string">
      Campaign identifier (if part of batch)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="campaigns" type="array">
  Active campaigns summary (when `includeDetails=true`):

  <Expandable title="campaign properties">
    <ResponseField name="campaignId" type="string">
      Campaign identifier
    </ResponseField>

    <ResponseField name="name" type="string">
      Campaign name
    </ResponseField>

    <ResponseField name="pending" type="number">
      Pending calls for this campaign
    </ResponseField>

    <ResponseField name="active" type="number">
      Active calls for this campaign
    </ResponseField>

    <ResponseField name="completed" type="number">
      Completed calls for this campaign
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X GET "https://api.talkos.io/api/calls/queue?includeDetails=true" \
  -H "x-api-key: your-api-key" \
  -H "x-tenant-id: your-tenant-id"
```

```javascript Node.js
const response = await fetch(
  'https://api.talkos.io/api/calls/queue?includeDetails=true',
  {
    method: 'GET',
    headers: {
      'x-api-key': 'your-api-key',
      'x-tenant-id': 'your-tenant-id'
    }
  }
);

const data = await response.json();
console.log(`Active: ${data.queue.active}, Pending: ${data.queue.pending}`);
console.log(`Capacity: ${data.queue.capacity.utilization}% utilized`);
```

```python Python
import requests

response = requests.get(
    'https://api.talkos.io/api/calls/queue',
    headers={
        'x-api-key': 'your-api-key',
        'x-tenant-id': 'your-tenant-id'
    },
    params={'includeDetails': True}
)

data = response.json()
print(f"Active: {data['queue']['active']}, Pending: {data['queue']['pending']}")
```
</CodeGroup>

## Notes

- Queue status updates in real-time
- Higher priority calls (9-10) are processed before lower priority calls
- Estimated wait time is calculated based on current capacity and average call duration
- Queue health indicators:
  - `healthy`: Utilization under 80%, no delays
  - `degraded`: Utilization 80-95%, slight delays expected
  - `overloaded`: Utilization over 95%, significant delays
- Consider monitoring queue status before starting large batch operations
- Rate limits reset every minute on the minute
