# Update Tenant

Updates the configuration and settings of an existing tenant. You can update the tenant name, email, provider configuration, webhooks, and metadata.

## Endpoint

```
PUT https://api.talkos.io/api/tenants/:tenantId
```

## Headers

<ParamField header="Content-Type" type="string" required default="application/json">
  Must be `application/json`
</ParamField>

<ParamField header="x-api-key" type="string" required>
  Your tenant API key.
  
  **Example:** `tk_live_abc123xyz789`
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  The tenant identifier.
  
  **Example:** `tenant_01HXYZ123456`
</ParamField>

## Path Parameters

<ParamField path="tenantId" type="string" required>
  The unique identifier of the tenant to update.
  
  **Example:** `tenant_01HXYZ123456`
</ParamField>

## Request Body

All fields are optional. Only include the fields you want to update.

<ParamField body="name" type="string">
  The updated display name for the tenant organization.
  
  - Minimum length: 2 characters
  - Maximum length: 100 characters
  
  **Example:** `"Acme Corp International"`
</ParamField>

<ParamField body="email" type="string">
  Updated primary contact email. Will trigger a verification email.
  
  **Example:** `"newemail@acme.com"`
</ParamField>

<ParamField body="providerConfig" type="object">
  Updated custom telephony provider configuration. Use this to enable or update the tenant's own provider credentials.
</ParamField>

<Expandable title="providerConfig properties">
  <ParamField body="providerConfig.useCustomProvider" type="boolean">
    Set to `true` to use the tenant's own provider credentials. Set to `false` to revert to TalkOS's default provider.
    
    **Example:** `true`
  </ParamField>

  <ParamField body="providerConfig.defaultProvider" type="string">
    The telephony provider to use. Required if `useCustomProvider` is `true`.
    
    **Allowed values:** `plivo`, `exotel`
  </ParamField>

  <ParamField body="providerConfig.plivo" type="object">
    Update Plivo configuration. Required if switching to Plivo.
  </ParamField>

  <Expandable title="providerConfig.plivo properties">
    <ParamField body="providerConfig.plivo.authId" type="string">
      Updated Plivo Auth ID.
      
      **Example:** `"MAXXXXXXXXXXXXXXXXXX"`
    </ParamField>

    <ParamField body="providerConfig.plivo.authToken" type="string">
      Updated Plivo Auth Token.
      
      **Example:** `"new_auth_token_xyz"`
    </ParamField>

    <ParamField body="providerConfig.plivo.fromNumber" type="string">
      Updated outbound caller phone number. Must be E.164 format.
      
      **Example:** `"+14155551234"`
    </ParamField>
  </Expandable>

  <ParamField body="providerConfig.exotel" type="object">
    Update Exotel configuration. Required if switching to Exotel.
  </ParamField>

  <Expandable title="providerConfig.exotel properties">
    <ParamField body="providerConfig.exotel.apiKey" type="string">
      Updated Exotel API Key.
    </ParamField>

    <ParamField body="providerConfig.exotel.apiToken" type="string">
      Updated Exotel API Token.
    </ParamField>

    <ParamField body="providerConfig.exotel.accountSid" type="string">
      Updated Exotel Account SID.
    </ParamField>

    <ParamField body="providerConfig.exotel.callerId" type="string">
      Updated ExoPhone number for outbound calls.
    </ParamField>

    <ParamField body="providerConfig.exotel.subdomain" type="string">
      Updated Exotel subdomain.
      
      **Allowed values:** `api.exotel.com`, `api.in.exotel.com`
    </ParamField>
  </Expandable>
</Expandable>

<ParamField body="webhookUrl" type="string">
  Updated webhook URL. Set to `null` to remove the webhook.
  
  **Example:** `"https://api.acme.com/webhooks/talkos-v2"`
</ParamField>

<ParamField body="webhookSecret" type="string">
  Secret for webhook signature verification. We'll include this in the `X-Webhook-Signature` header.
  
  **Example:** `"whsec_abc123xyz789"`
</ParamField>

<ParamField body="metadata" type="object">
  Updated custom metadata. This completely replaces existing metadata.
  
  **Example:** `{"internalId": "cust_12345", "tier": "premium"}`
</ParamField>

<ParamField body="settings" type="object">
  Tenant-level settings.
</ParamField>

<Expandable title="settings properties">
  <ParamField body="settings.callRecordingEnabled" type="boolean">
    Enable or disable call recording for all calls.
    
    **Default:** `true`
  </ParamField>

  <ParamField body="settings.transcriptionEnabled" type="boolean">
    Enable or disable automatic transcription.
    
    **Default:** `true`
  </ParamField>

  <ParamField body="settings.defaultLanguage" type="string">
    Default language for AI agents and transcription.
    
    **Example:** `"en-US"`
  </ParamField>

  <ParamField body="settings.timezone" type="string">
    Default timezone for reports and scheduling.
    
    **Example:** `"America/New_York"`
  </ParamField>

  <ParamField body="settings.dailyReportEmail" type="boolean">
    Whether to send daily usage reports via email.
    
    **Default:** `false`
  </ParamField>
</Expandable>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the update was successful.
</ResponseField>

<ResponseField name="data" type="object">
  The updated tenant object with all current values.
</ResponseField>

<Expandable title="data properties">
  <ResponseField name="data.id" type="string">
    The tenant's unique identifier.
  </ResponseField>

  <ResponseField name="data.name" type="string">
    The updated display name.
  </ResponseField>

  <ResponseField name="data.email" type="string">
    The current email address.
  </ResponseField>

  <ResponseField name="data.plan" type="string">
    The subscription plan (cannot be changed via this endpoint).
  </ResponseField>

  <ResponseField name="data.status" type="string">
    The current account status.
  </ResponseField>

  <ResponseField name="data.providerConfig" type="object">
    The current custom provider configuration (sensitive values masked). Only present if `useCustomProvider` is `true`.
  </ResponseField>

  <ResponseField name="data.webhookUrl" type="string">
    The current webhook URL.
  </ResponseField>

  <ResponseField name="data.metadata" type="object">
    The current metadata object.
  </ResponseField>

  <ResponseField name="data.settings" type="object">
    The current settings object.
  </ResponseField>

  <ResponseField name="data.updatedAt" type="string">
    ISO 8601 timestamp of this update.
  </ResponseField>

  <ResponseField name="data.updatedFields" type="array">
    List of fields that were modified in this update.
    
    **Example:** `["name", "webhookUrl", "config.plivo.authToken"]`
  </ResponseField>
</Expandable>

<ResponseField name="message" type="string">
  A human-readable success message.
</ResponseField>

## Notes

<Warning>
  **Credential Validation**: When updating provider credentials, TalkOS will attempt to validate them before saving. If validation fails, the update will be rejected.
</Warning>

<Info>
  **Plan Changes**: To change your subscription plan, use the billing portal or contact sales. Plan changes cannot be made via this endpoint.
</Info>

<Tip>
  **Metadata Replacement**: The `metadata` field is completely replaced on update. If you want to add a new field while keeping existing ones, first fetch the current metadata, merge your changes, and then update.
</Tip>

<Note>
  **Email Changes**: Changing the email address will trigger a verification email. The new email won't be active until verified.
</Note>
