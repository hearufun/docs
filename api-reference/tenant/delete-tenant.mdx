# Delete Tenant

Permanently delete a tenant account and all associated data. This action is irreversible and should be used with extreme caution.

<Warning>
This action is **permanent and irreversible**. All tenant data including call recordings, transcripts, analytics, and configuration will be permanently deleted. Consider using Suspend Tenant instead if you may need to restore access later.
</Warning>

## Endpoint

```
DELETE /api/tenants/:tenantId
```

## Headers

<ParamField header="x-api-key" type="string" required>
  Admin API key with tenant deletion permissions
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Path Parameters

<ParamField path="tenantId" type="string" required>
  Unique identifier of the tenant to delete
</ParamField>

## Request Body

<ParamField body="confirmation" type="string" required>
  Must be the exact string `DELETE_PERMANENTLY` to confirm the deletion. This prevents accidental deletions.
</ParamField>

<ParamField body="reason" type="string" required>
  Reason for deletion. Logged for audit purposes and compliance records.
  
  Common reasons:
  - `customer_request` - Tenant requested account deletion
  - `gdpr_request` - GDPR right to erasure request
  - `account_closure` - Business account closure
  - `data_retention_policy` - Automatic deletion per retention policy
  - `duplicate_account` - Removing duplicate tenant
  - `test_account` - Removing test/development account
</ParamField>

<ParamField body="deleteRecordings" type="boolean" default="true">
  Whether to delete all call recordings. If `false`, recordings will be anonymized but retained.
</ParamField>

<ParamField body="deleteAnalytics" type="boolean" default="true">
  Whether to delete analytics data. If `false`, analytics will be anonymized and retained for aggregate statistics.
</ParamField>

<ParamField body="exportDataFirst" type="boolean" default="false">
  Whether to export all tenant data before deletion. If `true`, an export will be generated and sent to the tenant's registered email before final deletion.
</ParamField>

<ParamField body="notifyTenant" type="boolean" default="true">
  Whether to send a deletion confirmation email to the tenant's registered email addresses.
</ParamField>

<ParamField body="deletionNote" type="string">
  Internal note about the deletion. Used for administrative tracking and compliance documentation.
</ParamField>

<ParamField body="scheduledDeletion" type="string">
  ISO 8601 timestamp to schedule deletion for a future date. If provided, deletion will be scheduled rather than immediate, allowing time for the tenant to be notified and potentially cancel.
</ParamField>

<ParamField body="gracePeriodDays" type="number" default="0">
  Number of days before final deletion. During this period, the tenant is marked for deletion but can be restored. Range: 0-30 days.
</ParamField>

  Whether the deletion was initiated successfully
</ResponseField>

<ResponseField name="deletionStatus" type="string">

## Example Request

<CodeGroup>
```bash cURL
curl -X DELETE "https://api.talkos.io/api/tenants/tenant_abc123" \
  -H "Content-Type: application/json" \
  -H "x-api-key: admin-api-key" \
  -d '{
    "confirmation": "DELETE_PERMANENTLY",
    "reason": "customer_request",
    "deleteRecordings": true,
    "deleteAnalytics": true,
    "exportDataFirst": true,
    "notifyTenant": true,
    "deletionNote": "Customer submitted account deletion request via support ticket #12345",
    "gracePeriodDays": 7
  }'
```

```javascript Node.js
const response = await fetch(
  'https://api.talkos.io/api/tenants/tenant_abc123',
  {
    method: 'DELETE',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': 'admin-api-key'
    },
    body: JSON.stringify({
      confirmation: 'DELETE_PERMANENTLY',
      reason: 'customer_request',
      deleteRecordings: true,
      deleteAnalytics: true,
      exportDataFirst: true,
      notifyTenant: true,
      deletionNote: 'Customer submitted account deletion request via support ticket #12345',
      gracePeriodDays: 7
    })
  }
);

const data = await response.json();
console.log(`Deletion status: ${data.deletionStatus}`);
```

```python Python
import requests

response = requests.delete(
    'https://api.talkos.io/api/tenants/tenant_abc123',
    headers={
        'Content-Type': 'application/json',
        'x-api-key': 'admin-api-key'
    },
    json={
        'confirmation': 'DELETE_PERMANENTLY',
        'reason': 'customer_request',
        'deleteRecordings': True,
        'deleteAnalytics': True,
        'exportDataFirst': True,
        'notifyTenant': True,
        'deletionNote': 'Customer submitted account deletion request via support ticket #12345',
        'gracePeriodDays': 7
    }
)

data = response.json()
print(f"Deletion status: {data['deletionStatus']}")
```
</CodeGroup>

## Restoring a Pending Deletion

If the tenant is within the grace period, you can restore it using the Activate Tenant endpoint:

```bash
curl -X POST "https://api.talkos.io/api/tenants/tenant_abc123/activate" \
  -H "Content-Type: application/json" \
  -H "x-api-key: admin-api-key" \
  -d '{
    "reason": "deletion_cancelled",
    "activationNote": "Customer cancelled deletion request"
  }'
```

## Notes

- **This action is irreversible** once the grace period ends
- All API keys associated with the tenant are immediately revoked
- Phone numbers are released and may be reassigned to other customers
- All recordings, transcripts, and data are permanently deleted from storage
- A compliance record is maintained for audit purposes (metadata only, no content)
- Data export includes: call records, recordings, transcripts, analytics, configuration
- Export URLs expire after 24 hours
- Large tenants with significant data may have deletion processed in the background
- Grace period allows accounts to be restored if deletion was accidental
- For GDPR compliance: use `reason: "gdpr_request"` to ensure proper documentation
- Active calls must be terminated before deletion can proceed
