# Activate Tenant

Activate a new tenant account or reactivate a previously suspended tenant, restoring full API access and call capabilities.

## Endpoint

```
POST /api/tenants/:tenantId/activate
```

## Headers

<ParamField header="x-api-key" type="string" required>
  Admin API key with tenant management permissions
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Path Parameters

<ParamField path="tenantId" type="string" required>
  Unique identifier of the tenant to activate
</ParamField>

## Request Body

<ParamField body="reason" type="string">
  Reason for activation/reactivation. Logged for audit purposes.
  
  Common reasons:
  - `payment_received` - Outstanding payment was received
  - `issue_resolved` - Policy violation or security issue resolved
  - `customer_request` - Tenant requested reactivation
  - `trial_started` - New trial account activation
  - `upgrade_completed` - Plan upgrade completed
  - `administrative` - Internal administrative action
</ParamField>

<ParamField body="notifyTenant" type="boolean" default="true">
  Whether to send an activation notification email to the tenant's registered email addresses.
</ParamField>

<ParamField body="activationNote" type="string">
  Internal note about the activation. Not visible to the tenant. Used for administrative tracking.
</ParamField>

<ParamField body="resetRateLimits" type="boolean" default="false">
  Whether to reset any rate limit counters that may have been hit before suspension.
</ParamField>

<ParamField body="clearWarnings" type="boolean" default="false">
  Whether to clear any warning flags on the account (e.g., low balance warnings).
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the activation was successful
</ResponseField>

<ResponseField name="tenant" type="object">
  Updated tenant information:

  <Expandable title="tenant properties">
    <ResponseField name="tenantId" type="string">
      Tenant identifier
    </ResponseField>

    <ResponseField name="name" type="string">
      Tenant name
    </ResponseField>

    <ResponseField name="status" type="string">
      New status: `active`
    </ResponseField>

    <ResponseField name="activatedAt" type="string">
      ISO 8601 timestamp of activation
    </ResponseField>

    <ResponseField name="previousStatus" type="string">
      Status before activation: `suspended`, `pending`, or `inactive`
    </ResponseField>

    <ResponseField name="suspendedDuration" type="number">
      If reactivating from suspension, number of seconds the account was suspended
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="capabilities" type="object">
  Restored capabilities:

  <Expandable title="capabilities properties">
    <ResponseField name="apiAccess" type="boolean">
      API access restored
    </ResponseField>

    <ResponseField name="outboundCalls" type="boolean">
      Outbound calling enabled
    </ResponseField>

    <ResponseField name="inboundCalls" type="boolean">
      Inbound calling enabled
    </ResponseField>

    <ResponseField name="batchCalling" type="boolean">
      Batch calling enabled
    </ResponseField>

    <ResponseField name="webhooks" type="boolean">
      Webhook delivery resumed
    </ResponseField>

    <ResponseField name="recordings" type="boolean">
      Recording access enabled
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="auditId" type="string">
  Audit log entry ID for this action
</ResponseField>

## Example Request

<CodeGroup>
```bash cURL
curl -X POST "https://api.talkos.io/api/tenants/tenant_abc123/activate" \
  -H "Content-Type: application/json" \
  -H "x-api-key: admin-api-key" \
  -d '{
    "reason": "payment_received",
    "notifyTenant": true,
    "activationNote": "Outstanding invoice #INV-2026-001 paid in full.",
    "resetRateLimits": true,
    "clearWarnings": true
  }'
```

```javascript Node.js
const response = await fetch(
  'https://api.talkos.io/api/tenants/tenant_abc123/activate',
  {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'x-api-key': 'admin-api-key'
    },
    body: JSON.stringify({
      reason: 'payment_received',
      notifyTenant: true,
      activationNote: 'Outstanding invoice #INV-2026-001 paid in full.',
      resetRateLimits: true,
      clearWarnings: true
    })
  }
);

const data = await response.json();
console.log(`Tenant activated: ${data.tenant.status}`);
```

```python Python
import requests

response = requests.post(
    'https://api.talkos.io/api/tenants/tenant_abc123/activate',
    headers={
        'Content-Type': 'application/json',
        'x-api-key': 'admin-api-key'
    },
    json={
        'reason': 'payment_received',
        'notifyTenant': True,
        'activationNote': 'Outstanding invoice #INV-2026-001 paid in full.',
        'resetRateLimits': True,
        'clearWarnings': True
    }
)

data = response.json()
print(f"Tenant activated: {data['tenant']['status']}")
```
</CodeGroup>

## Notes

- Activation takes effect immediately
- All API capabilities are restored upon activation
- Any paused webhook deliveries will resume
- Previously queued calls (before suspension) are NOT automatically resumed
- The tenant's credit balance is preserved through suspension/activation cycles
- Activation events are logged in the audit trail
- Rate limits are NOT automatically reset unless `resetRateLimits: true` is specified
- For new tenant activations, use this endpoint after creating the tenant
- Capabilities depend on the tenant's subscription plan
