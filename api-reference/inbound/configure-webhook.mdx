---
title: "Configure Inbound Webhook"
api: "POST https://api.talkos.io/api/inbound/config"
description: "Configure your tenant's inbound webhook to handle incoming calls with custom AI behavior."
---

Configure the webhook URL and fallback settings for handling inbound calls. When a call comes in, TalkOS will POST call details to your webhook to retrieve dynamic instructions.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your tenant identifier
</ParamField>

## Body Parameters

<ParamField body="webhookUrl" type="string" required>
  The URL TalkOS will call when an inbound call is received. Must be HTTPS and publicly accessible.
</ParamField>

<ParamField body="webhookTimeoutMs" type="integer" default={5000}>
  Maximum time in milliseconds to wait for webhook response before using fallback config. Range: 1000-30000.
</ParamField>

<ParamField body="fallbackConfig" type="object">
  Configuration used when webhook fails or times out.

  <Expandable title="fallbackConfig properties">
    <ParamField body="fallbackConfig.prompt" type="string">
      System prompt for the AI agent when fallback is activated.
    </ParamField>

    <ParamField body="fallbackConfig.greeting" type="string">
      Initial greeting message spoken to the caller.
    </ParamField>

    <ParamField body="fallbackConfig.maxDuration" type="integer" default={300}>
      Maximum call duration in seconds (1-3600).
    </ParamField>

    <ParamField body="fallbackConfig.language" type="string" default="en-US">
      Language code for speech recognition and synthesis.
    </ParamField>

    <ParamField body="fallbackConfig.voiceId" type="string">
      Voice identifier for text-to-speech. See /api/voices for available options.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="retryConfig" type="object">
  Webhook retry configuration.

  <Expandable title="retryConfig properties">
    <ParamField body="retryConfig.maxRetries" type="integer" default={2}>
      Number of retry attempts (0-5).
    </ParamField>

    <ParamField body="retryConfig.retryDelayMs" type="integer" default={500}>
      Delay between retries in milliseconds.
    </ParamField>
  </Expandable>
</ParamField>

## Webhook Request Format

When an inbound call is received, TalkOS sends a POST request to your webhook:

```json
{
  "event": "inbound_call",
  "callId": "call_abc123xyz",
  "from": "+14155551234",
  "to": "+18005559999",
  "timestamp": "2026-02-18T10:30:00.000Z",
  "metadata": {
    "callerIdName": "John Doe",
    "carrier": "Verizon"
  }
}
```

## Webhook Response Format

Your webhook should return call handling instructions:

```json
{
  "action": "answer",
  "prompt": "You are Sarah from Acme Corp customer support...",
  "greeting": "Hello! Thank you for calling Acme Corp. How can I help you?",
  "voiceId": "nova",
  "language": "en-US",
  "maxDuration": 600,
  "recordCall": true,
  "metadata": {
    "customerId": "cust_12345"
  }
}
```
