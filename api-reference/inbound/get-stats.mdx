---
title: "Get Inbound Statistics"
api: "GET https://api.talkos.io/api/inbound/stats"
description: "Retrieve detailed statistics and metrics for your inbound call operations."
---

Get comprehensive statistics about your inbound call handling including call volumes, success rates, performance metrics, and trends.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your tenant identifier
</ParamField>

## Query Parameters

<ParamField query="startDate" type="string">
  Start date for the statistics period (ISO 8601 format). Defaults to 30 days ago.
</ParamField>

<ParamField query="endDate" type="string">
  End date for the statistics period (ISO 8601 format). Defaults to now.
</ParamField>

<ParamField query="granularity" type="string" default="day">
  Time granularity for trend data. Options: `hour`, `day`, `week`, `month`.
</ParamField>

<ParamField query="phoneNumber" type="string">
  Filter statistics to a specific phone number (E.164 format).
</ParamField>

## Request

<CodeGroup>
```bash cURL
curl -X GET "https://api.talkos.io/api/inbound/stats?startDate=2026-02-01&granularity=day" \
  -H "x-api-key: your-api-key" \
  -H "x-tenant-id: your-tenant-id"
```

```javascript Node.js
const response = await fetch('https://api.talkos.io/api/inbound/stats?startDate=2026-02-01&granularity=day', {
  method: 'GET',
  headers: {
    'x-api-key': 'your-api-key',
    'x-tenant-id': 'your-tenant-id'
  }
});

const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.talkos.io/api/inbound/stats',
    headers={
        'x-api-key': 'your-api-key',
        'x-tenant-id': 'your-tenant-id'
    },
    params={
        'startDate': '2026-02-01',
        'granularity': 'day'
    }
)

data = response.json()
```
</CodeGroup>

## Response Fields

<ResponseField name="data.summary" type="object">
  Overall call statistics for the period.
</ResponseField>

<ResponseField name="data.summary.totalCalls" type="integer">
  Total number of inbound calls received.
</ResponseField>

<ResponseField name="data.summary.answerRate" type="number">
  Percentage of calls successfully answered.
</ResponseField>

<ResponseField name="data.summary.peakConcurrentCalls" type="integer">
  Maximum number of simultaneous calls during the period.
</ResponseField>

<ResponseField name="data.webhookMetrics" type="object">
  Performance metrics for your webhook endpoint.
</ResponseField>

<ResponseField name="data.webhookMetrics.successRate" type="number">
  Percentage of successful webhook responses.
</ResponseField>

<ResponseField name="data.webhookMetrics.p95LatencyMs" type="number">
  95th percentile webhook response time in milliseconds.
</ResponseField>

<ResponseField name="data.callOutcomes" type="object">
  Breakdown of how calls ended.
</ResponseField>

<ResponseField name="data.qualityMetrics" type="object">
  Call quality and sentiment analysis metrics.
</ResponseField>

<ResponseField name="data.trends" type="array">
  Time-series data based on selected granularity.
</ResponseField>

<ResponseField name="data.peakHours" type="array">
  Hours with highest call volume (0-23 in tenant timezone).
</ResponseField>

<Tip>
  Use `peakHours` data to optimize staffing for human escalations and ensure your webhook infrastructure can handle peak loads.
</Tip>
