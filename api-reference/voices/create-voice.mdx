## Overview

Create a custom voice by cloning from an audio sample or configuring voice parameters. Custom voices allow your AI agents to speak with a consistent, branded voice identity tailored to your use case.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your tenant identifier
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Body Parameters

<ParamField body="name" type="string" required>
  Name for the custom voice. Must be unique within your tenant.
</ParamField>

<ParamField body="description" type="string" required>
  Description of the voice characteristics and intended use case.
</ParamField>

<ParamField body="sampleUrl" type="string" required>
  URL to an audio file for voice cloning. The audio should be:
  - Clear speech with minimal background noise
  - 30 seconds to 5 minutes in length
  - Single speaker only
  - Supported formats: MP3, WAV, M4A, FLAC
</ParamField>

<ParamField body="language" type="string">
  Primary language of the voice. Default: `en-US`. See [List Languages](/api-reference/voices/list-languages) for supported options.
</ParamField>

<ParamField body="settings" type="object">
  Voice configuration settings
  <Expandable title="properties">
    <ParamField body="stability" type="number">
      Voice stability (0-1). Higher values produce more consistent output. Default: `0.75`.
    </ParamField>
    <ParamField body="similarityBoost" type="number">
      Similarity to original voice (0-1). Higher values sound more like the sample. Default: `0.75`.
    </ParamField>
    <ParamField body="style" type="number">
      Speaking style expressiveness (0-1). Higher values are more expressive. Default: `0.5`.
    </ParamField>
    <ParamField body="speakerBoost" type="boolean">
      Enable speaker clarity boost. Default: `true`.
    </ParamField>
  </Expandable>
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the voice was created successfully
</ResponseField>

<ResponseField name="voice" type="object">
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Unique identifier for the voice
    </ResponseField>
    <ResponseField name="name" type="string">
      Name of the voice
    </ResponseField>
    <ResponseField name="description" type="string">
      Voice description
    </ResponseField>
    <ResponseField name="type" type="string">
      Voice type: `cloned` or `preset`
    </ResponseField>
    <ResponseField name="language" type="string">
      Primary language code
    </ResponseField>
    <ResponseField name="status" type="string">
      Processing status: `processing`, `ready`, `failed`
    </ResponseField>
    <ResponseField name="previewUrl" type="string">
      URL to a preview audio sample (available when status is `ready`)
    </ResponseField>
    <ResponseField name="settings" type="object">
      Voice settings configuration
    </ResponseField>
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of creation
    </ResponseField>
  </Expandable>
</ResponseField>

## Voice Cloning Requirements

<Note>
Voice cloning typically takes 1-3 minutes to process. The voice will have a status of `processing` initially and will update to `ready` once complete.
</Note>

<Tip>
**For best cloning results:**
- Use high-quality audio (44.1kHz or higher sample rate)
- Ensure clean speech without music or background noise
- Include varied intonation and natural speech patterns
- Avoid whispered or shouted speech
</Tip>

<Warning>
You must have rights to use the voice sample. Do not clone voices without proper authorization from the voice owner.
</Warning>
