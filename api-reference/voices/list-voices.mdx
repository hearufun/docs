description: "Retrieve all available voices including custom and preset voices"
---

## Overview

List all voices available to your tenant, including custom cloned voices and TalkOS preset voices. Use this endpoint to discover voice options for your AI agents.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your tenant identifier
</ParamField>

## Query Parameters

<ParamField query="type" type="string">
  Filter by voice type. Options: `all`, `cloned`, `preset`. Default: `all`.
</ParamField>

<ParamField query="language" type="string">
  Filter by language code (e.g., `en-US`, `es-ES`). Returns voices supporting this language.
</ParamField>

<ParamField query="status" type="string">
  Filter by status. Options: `all`, `ready`, `processing`, `failed`. Default: `ready`.
</ParamField>

<ParamField query="page" type="number">
  Page number for pagination. Default: `1`.
</ParamField>

<ParamField query="limit" type="number">
  Results per page. Default: `20`. Maximum: `100`.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request completed successfully
</ResponseField>

<ResponseField name="voices" type="array">
  Array of voice objects
  <Expandable title="voice object">
    <ResponseField name="id" type="string">
      Unique identifier for the voice
    </ResponseField>
    <ResponseField name="name" type="string">
      Display name of the voice
    </ResponseField>
    <ResponseField name="description" type="string">
      Description of voice characteristics
    </ResponseField>
    <ResponseField name="type" type="string">
      Voice type: `cloned` (custom) or `preset` (built-in)
    </ResponseField>
    <ResponseField name="language" type="string">
      Primary language code
    </ResponseField>
    <ResponseField name="supportedLanguages" type="array">
      All languages the voice supports
    </ResponseField>
    <ResponseField name="gender" type="string">
      Voice gender: `male`, `female`, `neutral`
    </ResponseField>
    <ResponseField name="age" type="string">
      Voice age range: `young`, `middle`, `mature`
    </ResponseField>
    <ResponseField name="accent" type="string">
      Voice accent (e.g., `american`, `british`, `australian`)
    </ResponseField>
    <ResponseField name="previewUrl" type="string">
      URL to preview audio sample
    </ResponseField>
    <ResponseField name="status" type="string">
      Status: `ready`, `processing`, `failed`
    </ResponseField>
    <ResponseField name="isDefault" type="boolean">
      Whether this is a preset voice
    </ResponseField>
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp (null for preset voices)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="pagination" type="object">
  <Expandable title="properties">
    <ResponseField name="page" type="number">
      Current page number
    </ResponseField>
    <ResponseField name="limit" type="number">
      Results per page
    </ResponseField>
    <ResponseField name="totalPages" type="number">
      Total number of pages
    </ResponseField>
    <ResponseField name="totalCount" type="number">
      Total number of voices
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash cURL
curl -X GET "https://api.talkos.io/api/voices" \
  -H "x-api-key: your-api-key" \
  -H "x-tenant-id: your-tenant-id"
```

```bash cURL (Filter by Type)
curl -X GET "https://api.talkos.io/api/voices?type=cloned" \
  -H "x-api-key: your-api-key" \
  -H "x-tenant-id: your-tenant-id"
```

```bash cURL (Filter by Language)
curl -X GET "https://api.talkos.io/api/voices?language=es-ES" \
  -H "x-api-key: your-api-key" \
  -H "x-tenant-id: your-tenant-id"
```
</RequestExample>

## Voice Categories

| Type | Description |
|------|-------------|
| `preset` | Built-in TalkOS voices, professionally recorded and optimized |
| `cloned` | Custom voices created by cloning audio samples |

## Usage with Agents

To assign a voice to an AI agent, use the voice `id` when [creating](/api-reference/agents/create) or [updating](/api-reference/agents/update) an agent configuration.
