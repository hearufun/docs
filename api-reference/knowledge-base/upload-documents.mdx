description: "Upload documents to a knowledge base for indexing and semantic search"
---

## Overview

Upload documents to an existing knowledge base. Documents are automatically parsed, chunked, and indexed for semantic search. Your AI agents can then retrieve relevant information from these documents during calls.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your tenant identifier
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `multipart/form-data`
</ParamField>

## Form Data Parameters

<ParamField body="file" type="file" required>
  The document file to upload. Maximum file size: 50MB.
  
  **Supported formats:**
  - PDF (`.pdf`) - Standard PDF documents
  - Text (`.txt`) - Plain text files
  - Markdown (`.md`) - Markdown formatted documents
  - Word (`.docx`) - Microsoft Word documents
</ParamField>

<ParamField body="knowledgeBaseId" type="string" required>
  The ID of the knowledge base to upload the document to.
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the document was uploaded and indexed successfully
</ResponseField>

<ResponseField name="document" type="object">
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Unique identifier for the uploaded document
    </ResponseField>
    <ResponseField name="filename" type="string">
      Original filename of the uploaded document
    </ResponseField>
    <ResponseField name="fileType" type="string">
      MIME type of the document
    </ResponseField>
    <ResponseField name="fileSize" type="number">
      Size of the file in bytes
    </ResponseField>
    <ResponseField name="chunkCount" type="number">
      Number of chunks the document was split into
    </ResponseField>
    <ResponseField name="tokenCount" type="number">
      Total tokens indexed from the document
    </ResponseField>
    <ResponseField name="status" type="string">
      Processing status: `processing`, `indexed`, `failed`
    </ResponseField>
    <ResponseField name="createdAt" type="string">
      ISO 8601 timestamp of upload
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="knowledgeBase" type="object">
  <Expandable title="properties">
    <ResponseField name="id" type="string">
      Knowledge base ID
    </ResponseField>
    <ResponseField name="documentCount" type="number">
      Updated total document count
    </ResponseField>
    <ResponseField name="totalTokens" type="number">
      Updated total tokens indexed
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash cURL
curl -X POST https://api.talkos.io/api/knowledge-base/upload \
  -H "x-api-key: your-api-key" \
  -H "x-tenant-id: your-tenant-id" \
  -F "file=@/path/to/document.pdf" \
  -F "knowledgeBaseId=kb_8f3d2a1b-4c5e-6f7a-8b9c-0d1e2f3a4b5c"
```

```bash cURL (Multiple Files)
# Upload multiple documents sequentially
for file in docs/*.pdf; do
  curl -X POST https://api.talkos.io/api/knowledge-base/upload \
    -H "x-api-key: your-api-key" \
    -H "x-tenant-id: your-tenant-id" \
    -F "file=@$file" \
    -F "knowledgeBaseId=kb_8f3d2a1b-4c5e-6f7a-8b9c-0d1e2f3a4b5c"
done
```
</RequestExample>

## Processing Notes

<Note>
Large documents may take a few seconds to process. The initial response will show `status: "processing"` and will update to `"indexed"` once complete. You can check the status using the [Get Stats](/api-reference/knowledge-base/get-stats) endpoint.
</Note>

<Warning>
Uploading a file with the same name as an existing document will create a new version. The old version will be marked as archived but still searchable unless explicitly deleted.
</Warning>
