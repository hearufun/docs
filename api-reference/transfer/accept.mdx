---
title: 'Accept Transfer'

description: 'Accept an incoming call transfer request'
---

## Overview

Accepts a pending call transfer, connecting the agent to the caller. This endpoint is typically called by the agent's client application when they choose to accept an incoming transferred call.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your tenant identifier
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`
</ParamField>

## Path Parameters

<ParamField path="callUuid" type="string" required>
  The unique identifier of the call being transferred
</ParamField>

## Body Parameters

<ParamField body="agentId" type="string" required>
  The ID of the agent accepting the transfer
</ParamField>

<ParamField body="acknowledgeContext" type="boolean" default="false">
  Indicates if the agent has reviewed the transfer context
</ParamField>

<ParamField body="notes" type="string">
  Optional notes from the agent upon accepting
</ParamField>

</ResponseField>

<ResponseField name="agent" type="object">

  <Expandable title="properties">

## Workflow

1. Agent receives transfer notification via WebSocket or webhook
2. Agent reviews transfer context and caller information
3. Agent calls this endpoint to accept the transfer
4. System connects the agent to the caller
5. AI assistant disconnects from the call

## Error Codes

| Code | Description |
|------|-------------|
| `TRANSFER_NOT_FOUND` | No pending transfer exists for this call UUID |
| `TRANSFER_EXPIRED` | Transfer request timed out before acceptance |
| `TRANSFER_ALREADY_PROCESSED` | Transfer was already accepted or rejected |
| `AGENT_MISMATCH` | Agent ID doesn't match the transfer target |
| `AGENT_NOT_FOUND` | Agent ID does not exist |
| `CALL_ENDED` | The call has ended during the transfer process |
