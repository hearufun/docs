---
title: "Get Call Sentiment Analysis"
api: "GET https://api.talkos.io/api/analytics/sentiment/{callUuid}"
description: "Retrieve detailed per-turn sentiment analysis for a specific call"
---

## Overview

Get granular sentiment analysis for a specific call, including per-turn sentiment scores, emotion detection, and conversation flow analysis. This endpoint provides deep insights into how sentiment evolved throughout the conversation.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your tenant identifier
</ParamField>

## Path Parameters

<ParamField path="callUuid" type="string" required>
  The unique identifier of the call to analyze
</ParamField>

## Query Parameters

<ParamField query="includeTranscript" type="boolean" default="false">
  Include the transcript text for each turn
</ParamField>

<ParamField query="includeEmotions" type="boolean" default="true">
  Include detailed emotion breakdown for each turn
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="object">
  <Expandable title="properties">
    <ResponseField name="callUuid" type="string">
      The call identifier
    </ResponseField>
    <ResponseField name="overallSentiment" type="string">
      Overall call sentiment: `positive`, `neutral`, `negative`
    </ResponseField>
    <ResponseField name="overallScore" type="number">
      Overall sentiment score (-1 to 1)
    </ResponseField>
    <ResponseField name="sentimentJourney" type="string">
      Summary of sentiment progression: `improving`, `stable`, `declining`, `volatile`
    </ResponseField>
    <ResponseField name="turns" type="array">
      Per-turn sentiment analysis
      <Expandable title="turn object">
        <ResponseField name="turnIndex" type="integer">
          Sequential index of the conversation turn (0-based)
        </ResponseField>
        <ResponseField name="speaker" type="string">
          Speaker identifier: `customer` or `agent`
        </ResponseField>
        <ResponseField name="timestamp" type="string">
          Timestamp of the turn
        </ResponseField>
        <ResponseField name="duration" type="number">
          Duration of the turn in seconds
        </ResponseField>
        <ResponseField name="sentiment" type="string">
          Turn sentiment: `positive`, `neutral`, `negative`
        </ResponseField>
        <ResponseField name="sentimentScore" type="number">
          Sentiment score (-1 to 1)
        </ResponseField>
        <ResponseField name="emotions" type="object">
          Detected emotions with confidence scores (0-1)
          <Expandable title="emotion scores">
            <ResponseField name="joy" type="number">Confidence for joy</ResponseField>
            <ResponseField name="anger" type="number">Confidence for anger</ResponseField>
            <ResponseField name="frustration" type="number">Confidence for frustration</ResponseField>
            <ResponseField name="satisfaction" type="number">Confidence for satisfaction</ResponseField>
            <ResponseField name="confusion" type="number">Confidence for confusion</ResponseField>
            <ResponseField name="urgency" type="number">Confidence for urgency</ResponseField>
          </Expandable>
        </ResponseField>
        <ResponseField name="transcript" type="string">
          Transcript text (if `includeTranscript=true`)
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="keyMoments" type="array">
      Significant sentiment shifts during the call
      <Expandable title="moment object">
        <ResponseField name="turnIndex" type="integer">
          Turn where the moment occurred
        </ResponseField>
        <ResponseField name="type" type="string">
          Type: `peak_positive`, `peak_negative`, `major_shift`, `resolution`
        </ResponseField>
        <ResponseField name="description" type="string">
          Human-readable description of the moment
        </ResponseField>
        <ResponseField name="scoreDelta" type="number">
          Sentiment score change
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="summary" type="object">
      <Expandable title="properties">
        <ResponseField name="startSentiment" type="number">
          Sentiment score at call start
        </ResponseField>
        <ResponseField name="endSentiment" type="number">
          Sentiment score at call end
        </ResponseField>
        <ResponseField name="lowestPoint" type="number">
          Lowest sentiment score during call
        </ResponseField>
        <ResponseField name="highestPoint" type="number">
          Highest sentiment score during call
        </ResponseField>
        <ResponseField name="volatility" type="number">
          Standard deviation of sentiment scores (higher = more volatile)
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

<RequestExample>
```bash cURL
curl -X GET "https://api.talkos.io/api/analytics/sentiment/call_8f2a9c4e-1b3d-4e5f-a6b7-c8d9e0f1a2b3?includeTranscript=true&includeEmotions=true" \
  -H "x-api-key: your-api-key" \
  -H "x-tenant-id: your-tenant-id"
```

```javascript JavaScript
const callUuid = 'call_8f2a9c4e-1b3d-4e5f-a6b7-c8d9e0f1a2b3';

const params = new URLSearchParams({
  includeTranscript: 'true',
  includeEmotions: 'true'
});

const response = await fetch(
  `https://api.talkos.io/api/analytics/sentiment/${callUuid}?${params}`,
  {
    method: 'GET',
    headers: {
      'x-api-key': 'your-api-key',
      'x-tenant-id': 'your-tenant-id'
    }
  }
);

const data = await response.json();
```

```python Python
import requests

call_uuid = 'call_8f2a9c4e-1b3d-4e5f-a6b7-c8d9e0f1a2b3'

response = requests.get(
    f'https://api.talkos.io/api/analytics/sentiment/{call_uuid}',
    params={
        'includeTranscript': True,
        'includeEmotions': True
    },
    headers={
        'x-api-key': 'your-api-key',
        'x-tenant-id': 'your-tenant-id'
    }
)

data = response.json()
```

## Sentiment Journey Types

<CardGroup cols={2}>
  <Card title="Improving" icon="arrow-trend-up">
    Call started with lower sentiment but ended positively. Often indicates successful issue resolution.
  </Card>
  <Card title="Stable" icon="minus">
    Sentiment remained consistent throughout the call. Common in routine inquiries.
  </Card>
  <Card title="Declining" icon="arrow-trend-down">
    Sentiment deteriorated during the call. May indicate escalation needed.
  </Card>
  <Card title="Volatile" icon="chart-line">
    Significant sentiment swings throughout. Often seen in complex issue discussions.
  </Card>
</CardGroup>

## Use Cases

<AccordionGroup>
  <Accordion title="Quality Assurance">
    Review sentiment analysis for calls flagged for QA review. Identify specific moments where agents handled difficult situations well or poorly.
  </Accordion>
  <Accordion title="Training Material">
    Use calls with "improving" sentiment journeys as training examples. The `keyMoments` data highlights effective de-escalation techniques.
  </Accordion>
  <Accordion title="Customer Follow-up">
    Identify calls that ended with declining sentiment for proactive customer follow-up to prevent churn.
  </Accordion>
</AccordionGroup>

<Note>
  Sentiment analysis is processed asynchronously after call completion. Results are typically available within 2-5 minutes after the call ends.
</Note>

## Rate Limits

- **Standard tier**: 60 requests per minute
- **Enterprise tier**: 300 requests per minute
