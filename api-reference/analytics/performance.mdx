---

## Overview

Get comprehensive performance analytics for your AI agents. This endpoint provides key metrics to evaluate agent efficiency, effectiveness, and areas for improvement.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your tenant identifier
</ParamField>

## Query Parameters

<ParamField query="startDate" type="string" required>
  Start date for the query range in ISO 8601 format
</ParamField>

<ParamField query="endDate" type="string" required>
  End date for the query range in ISO 8601 format
</ParamField>

<ParamField query="agentId" type="string">
  Filter by specific agent ID (optional, returns all agents if not specified)
</ParamField>

<ParamField query="direction" type="string">
  Filter by call direction. Options: `inbound`, `outbound`
</ParamField>

<ParamField query="groupBy" type="string" default="day">
  Group results by time period. Options: `hour`, `day`, `week`, `month`
</ParamField>

<RequestExample>
```bash cURL
curl -X GET "https://api.talkos.io/api/analytics/performance?startDate=2026-02-01T00:00:00Z&endDate=2026-02-18T23:59:59Z&groupBy=day" \
  -H "x-api-key: your-api-key" \
  -H "x-tenant-id: your-tenant-id"
```

```javascript JavaScript
const params = new URLSearchParams({
  startDate: '2026-02-01T00:00:00Z',
  endDate: '2026-02-18T23:59:59Z',
  groupBy: 'day'
});

const response = await fetch(`https://api.talkos.io/api/analytics/performance?${params}`, {
  method: 'GET',
  headers: {
    'x-api-key': 'your-api-key',
    'x-tenant-id': 'your-tenant-id'
  }
});

const data = await response.json();
```

```python Python
import requests

response = requests.get(

        'startDate': '2026-02-01T00:00:00Z',

        'groupBy': 'day'
    },
    headers={
        'x-api-key': 'your-api-key',
        'x-tenant-id': 'your-tenant-id'
    }
)

data = response.json()
```

## Key Metrics Explained

<CardGroup cols={2}>
  <Card title="First Call Resolution (FCR)" icon="check-circle">
    Percentage of calls resolved without requiring a follow-up call. Higher FCR indicates effective issue resolution.
  </Card>
  <Card title="Transfer Rate" icon="arrow-right-arrow-left">
    Percentage of calls escalated to human agents. Lower rates indicate better AI agent capability.
  </Card>
  <Card title="Average Handle Time" icon="clock">
    Total time from call start to completion including any wrap-up. Key efficiency metric.
  </Card>
  <Card title="Abandonment Rate" icon="phone-slash">
    Percentage of callers who hung up before connecting. High rates may indicate capacity issues.
  </Card>
</CardGroup>

## Best Practices

<AccordionGroup>
  <Accordion title="Reduce Transfer Rate">
    Monitor `transferReasons` to identify gaps in AI agent training. High "complex_technical_issue" transfers may indicate need for expanded knowledge base.
  </Accordion>
  <Accordion title="Optimize Handle Time">
    Compare `avgCallDuration` across agents to identify efficiency patterns. Share best practices from high-performing agents.
  </Accordion>
  <Accordion title="Track Trends">
    Use time-series data to correlate performance changes with system updates, training changes, or external factors.
  </Accordion>
</AccordionGroup>

## Benchmarks

| Metric | Good | Excellent |
|--------|------|-----------|
| Transfer Rate | < 15% | < 8% |
| First Call Resolution | > 80% | > 90% |
| Avg Response Time | < 2s | < 1s |
| Abandonment Rate | < 5% | < 2% |
| Avg Sentiment Score | > 0.3 | > 0.5 |

<Note>
  Benchmarks are based on industry averages for AI-powered call centers. Your targets may vary based on use case complexity.
</Note>

## Rate Limits

- **Standard tier**: 30 requests per minute
- **Enterprise tier**: 120 requests per minute
