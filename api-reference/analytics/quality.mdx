## Overview

Get comprehensive call quality analytics to monitor and optimize the voice experience. This endpoint provides technical quality metrics including latency, audio quality, and conversation flow indicators.

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your tenant identifier
</ParamField>

## Query Parameters

<ParamField query="startDate" type="string" required>
  Start date for the query range in ISO 8601 format
</ParamField>

<ParamField query="endDate" type="string" required>
  End date for the query range in ISO 8601 format
</ParamField>

<ParamField query="groupBy" type="string" default="day">
  Group results by time period. Options: `hour`, `day`, `week`
</ParamField>

<ParamField query="direction" type="string">
  Filter by call direction. Options: `inbound`, `outbound`
</ParamField>

## Understanding Quality Metrics

<CardGroup cols={2}>
  <Card title="Latency" icon="gauge">
    End-to-end response time from customer speech to AI response. Target: under 500ms for natural conversation flow.
  </Card>
  <Card title="Audio Quality (MOS)" icon="volume-high">
    Mean Opinion Score (1-5 scale). Measures perceived audio quality. Target: above 4.0 for excellent quality.
  </Card>
  <Card title="Interruption Rate" icon="comment-slash">
    Percentage of conversational turns with overlapping speech. High rates may indicate latency issues or aggressive barge-in settings.
  </Card>
  <Card title="Silence Rate" icon="volume-xmark">
    Percentage of awkward pauses. High rates may indicate slow LLM responses or processing issues.
  </Card>
</CardGroup>

## Latency Targets

| Component | Target | Acceptable | Warning |
|-----------|--------|------------|---------|
| STT | < 100ms | < 150ms | > 200ms |
| LLM | < 200ms | < 350ms | > 500ms |
| TTS | < 75ms | < 100ms | > 150ms |
| Total | < 400ms | < 600ms | > 800ms |

## MOS Quality Scale

| Score | Rating | Experience |
|-------|--------|------------|
| 4.3+ | Excellent | Indistinguishable from being in-person |
| 4.0-4.3 | Good | Minor imperfections, good clarity |
| 3.5-4.0 | Fair | Noticeable issues, still usable |
| 3.0-3.5 | Poor | Significant quality issues |
| < 3.0 | Bad | Difficult to understand, frustrating |

## Optimization Tips

<AccordionGroup>
  <Accordion title="Reduce LLM Latency">
    LLM typically accounts for 50% of total latency. Optimize by:
    - Using streaming responses
    - Reducing prompt complexity
    - Enabling response caching
    - Using faster model variants
  </Accordion>
  <Accordion title="Improve Audio Quality">
    Address low MOS scores by:
    - Ensuring stable network connections
    - Using higher quality codecs
    - Implementing proper echo cancellation
    - Monitoring carrier quality
  </Accordion>
  <Accordion title="Reduce Interruptions">
    Lower interruption rates by:
    - Tuning barge-in sensitivity
    - Increasing end-of-speech detection threshold
    - Adding natural pauses in TTS output
  </Accordion>
  <Accordion title="Handle Silence Gracefully">
    Reduce awkward silences by:
    - Implementing filler phrases during processing
    - Using progressive response delivery
    - Setting appropriate timeout thresholds
  </Accordion>
</AccordionGroup>

<Warning>
  Consistently high latency (>800ms) significantly degrades conversation quality and customer satisfaction. Set up alerts to catch latency issues before they impact users.
</Warning>
