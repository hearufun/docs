# Search Chat Messages

Search across all chat messages with powerful filtering and full-text search capabilities. Find specific conversations, keywords, or patterns across your chat history.

## Endpoint

```
GET /api/features/chat/search
```

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your unique tenant identifier
</ParamField>

## Query Parameters

<ParamField query="query" type="string" required>
  Search query string (supports full-text search)
</ParamField>

<ParamField query="startDate" type="string">
  Filter by start date (ISO 8601 format)
</ParamField>

<ParamField query="endDate" type="string">
  Filter by end date (ISO 8601 format)
</ParamField>

<ParamField query="agentId" type="string">
  Filter by specific agent ID
</ParamField>

<ParamField query="channel" type="string">
  Filter by chat channel: `web`, `sms`, `email`, `whatsapp`
</ParamField>

<ParamField query="senderType" type="string">
  Filter by sender type: `agent`, `customer`, `bot`, `system`
</ParamField>

<ParamField query="limit" type="integer" default={20}>
  Maximum number of results to return (1-100)
</ParamField>

<ParamField query="offset" type="integer" default={0}>
  Number of results to skip for pagination
</ParamField>

<ParamField query="sortBy" type="string" default="relevance">
  Sort results by: `relevance`, `date_asc`, `date_desc`
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the search was performed successfully
</ResponseField>

<ResponseField name="query" type="string">
  The search query that was executed
</ResponseField>

<ResponseField name="results" type="array">
  Array of matching messages with context
</ResponseField>

<ResponseField name="totalResults" type="integer">
  Total number of matching results
</ResponseField>

<ResponseField name="pagination" type="object">
  Pagination information
</ResponseField>

<RequestExample>
```bash cURL
curl -X GET "https://api.talkos.io/api/features/chat/search?query=billing+issue&startDate=2026-02-01&limit=20&sortBy=relevance" \
  -H "x-api-key: your-api-key" \
  -H "x-tenant-id: your-tenant-id"
```

```javascript JavaScript
const params = new URLSearchParams({
  query: 'billing issue',
  startDate: '2026-02-01',
  limit: '20',
  sortBy: 'relevance'
});

const response = await fetch(`https://api.talkos.io/api/features/chat/search?${params}`, {
  headers: {
    'x-api-key': 'your-api-key',
    'x-tenant-id': 'your-tenant-id'
  }
});

const data = await response.json();
```

```python Python
import requests

response = requests.get(
    'https://api.talkos.io/api/features/chat/search',
    headers={
        'x-api-key': 'your-api-key',
        'x-tenant-id': 'your-tenant-id'
    },
    params={
        'query': 'billing issue',
        'startDate': '2026-02-01',
        'limit': 20,
        'sortBy': 'relevance'
    }
)

data = response.json()
```

## Error Codes

| Code | Description |
|------|-------------|
| `MISSING_QUERY` | Search query parameter is required |
| `INVALID_DATE_RANGE` | Start date must be before end date |
| `SEARCH_FAILED` | Search operation failed |
| `UNAUTHORIZED` | Invalid API key or tenant ID |

## Search Tips

- Use quotes for exact phrase matching: `"billing issue"`
- Use `AND`, `OR` for boolean operators: `billing AND refund`
- Use `-` to exclude terms: `billing -refund`
- Wildcards are supported: `bill*` matches billing, billed, etc.
