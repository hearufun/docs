# Start Chat Session

Initiate a chat session associated with an active call. This enables parallel text-based communication alongside the voice call, allowing agents to share links, documents, or follow-up information.

## Endpoint

```
POST /api/features/chat/:callUuid/start
```

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your unique tenant identifier
</ParamField>

## Path Parameters

<ParamField path="callUuid" type="string" required>
  Unique identifier of the active call
</ParamField>

## Request Body

<ParamField body="agentId" type="string" required>
  ID of the agent initiating the chat
</ParamField>

<ParamField body="customerContact" type="object">
  Customer contact information for chat delivery
</ParamField>

<ParamField body="customerContact.email" type="string">
  Customer email address
</ParamField>

<ParamField body="customerContact.phone" type="string">
  Customer phone number for SMS chat
</ParamField>

<ParamField body="channel" type="string" default="web">
  Chat channel: `web`, `sms`, `email`, `whatsapp`
</ParamField>

<ParamField body="sendGreeting" type="boolean" default={true}>
  Automatically send configured greeting message
</ParamField>

<ParamField body="metadata" type="object">
  Additional metadata to attach to the chat session
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  Whether the chat session was started successfully
</ResponseField>

<ResponseField name="chatId" type="string">
  Unique identifier for the chat session
</ResponseField>

<ResponseField name="callUuid" type="string">
  The associated call UUID
</ResponseField>

<ResponseField name="agentId" type="string">
  ID of the agent who initiated the chat
</ResponseField>

<ResponseField name="channel" type="string">
  The channel being used for the chat
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the chat: `active`, `pending`
</ResponseField>

<ResponseField name="chatUrl" type="string">
  URL for the customer to access the web chat (if applicable)
</ResponseField>

<ResponseField name="startedAt" type="string">
  ISO 8601 timestamp when the chat session started
</ResponseField>

## Error Codes

| Code | Description |
|------|-------------|
| `CALL_NOT_FOUND` | The specified call UUID does not exist |
| `CALL_NOT_ACTIVE` | The call is not currently in progress |
| `CHAT_EXISTS` | A chat session already exists for this call |
| `AGENT_NOT_FOUND` | The specified agent ID is invalid |
| `INVALID_CHANNEL` | The specified channel is not supported |
| `CHAT_DISABLED` | Chat feature is not enabled for this tenant |
| `UNAUTHORIZED` | Invalid API key or tenant ID |
