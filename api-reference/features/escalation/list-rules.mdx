# List Escalation Rules

Retrieve all configured escalation rules for your tenant, including both built-in and custom rules.

## Endpoint

```
GET /api/features/escalation/rules
```

## Headers

<ParamField header="x-api-key" type="string" required>
  Your TalkOS API key for authentication
</ParamField>

<ParamField header="x-tenant-id" type="string" required>
  Your unique tenant identifier
</ParamField>

## Response

<ResponseField name="rules" type="array">
  Array of all escalation rules
</ResponseField>

<ResponseField name="rules[].id" type="string">
  Unique identifier for the rule
</ResponseField>

<ResponseField name="rules[].name" type="string">
  Human-readable name of the rule
</ResponseField>

<ResponseField name="rules[].description" type="string">
  Description of what the rule does
</ResponseField>

<ResponseField name="rules[].conditions" type="array">
  Array of conditions that trigger the rule
</ResponseField>

<ResponseField name="rules[].logic" type="string">
  Logic operator for multiple conditions: `AND` or `OR`
</ResponseField>

<ResponseField name="rules[].priority" type="string">
  Priority level: `low`, `medium`, `high`, or `critical`
</ResponseField>

<ResponseField name="rules[].action" type="object">
  Action to take when rule triggers
</ResponseField>

<ResponseField name="rules[].cooldownSeconds" type="number">
  Cooldown period before rule can re-trigger
</ResponseField>

<ResponseField name="rules[].enabled" type="boolean">
  Whether the rule is currently active
</ResponseField>

<ResponseField name="rules[].builtIn" type="boolean">
  Whether this is a built-in system rule
</ResponseField>

<ResponseField name="count" type="number">
  Total number of rules
</ResponseField>
